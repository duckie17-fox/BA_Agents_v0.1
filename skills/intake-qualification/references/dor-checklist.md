# DoR Checklist

Mark each: ✅ Pass / ⚠️ Partial / ❌ Fail

| # | Criterion | Pass condition |
|---|-----------|---------------|
| 1 | Requester identified | Know who sent this and their role |
| 2 | Problem or need described | Derived from clarify step — hypothesis confirmed or root type identified; NOT copied from raw input |
| 3 | Affected users identified | Know which user segment is impacted |
| 4 | Business impact stated | Revenue, efficiency, risk, or customer retention mentioned |
| 5 | Scope roughly understood | Derived from clarify step — branching question confirmed scope signal; NOT guessed from raw input |
| 6 | Not a duplicate | No obvious overlap with known backlog items |
| 7 | Feasibility not blocked | No known hard blocker (legal, infra, dependency) |

**Result:**
- 6-7 Pass → ✅ Ready to proceed
- 4-5 Pass → ⚠️ Conditionally ready — proceed with noted gaps
- < 4 Pass → ❌ Not ready — return to requester with specific questions
